# Refact v8.2.3

Refact v8.2.3 completes the "Connection Problem" fix from v8.2.2 for **VS Code**. v8.2.2 restored the JetBrains in-IDE chat, but VS Code webviews still could not reach the local daemon: the daemon only trusted `vscode-webview://` origins whose authority looked like a hex UUID (`vscode-webview://9a0315fe-…`), while VS Code actually sends a base32-style lowercase authority on the wire (for example `vscode-webview://0ddd89pf9tgcs7jvdpe0qs2lctosmhsfsbhj2q7ncq6jv6cq2us4`). The daemon returned no `Access-Control-Allow-Origin` header, the browser blocked every request with a CORS error, and the sidebar showed "Connection Problem / Backend server unreachable" even though the daemon, worker, and project proxy were all healthy.

> [!IMPORTANT]
> **Early preview**
> The daemon and TUI are early preview and may be buggy. IDE plugins and the engine are the stable surfaces.

## Fixes

- Daemon auth/CORS: accept the real VS Code webview origin format. The `vscode-webview://` authority check now allows any alphanumeric-plus-dash authority instead of hex-only, covering both the persisted hex-UUID form and the base32-encoded form VS Code actually puts in the `Origin` header. The trust scope is unchanged otherwise: loopback peers only, JetBrains `http://refactai` unchanged, `null`/remote/unknown web origins still rejected, non-loopback access still requires authentication.

## Upgrade notes

- JetBrains users on v8.2.2 are unaffected by the bug; upgrading keeps behavior identical.
- VS Code users on v8.2.1/v8.2.2: update the extension and reload the window. The plugin auto-upgrades the local daemon to 8.2.3 on activation; if the sidebar still shows "Connection Problem", run `Developer: Reload Window` once so the extension re-attaches to the upgraded daemon.

## Install

Unix/macOS:

```sh
curl -fsSL https://raw.githubusercontent.com/JegernOUTT/refact/main/install.sh | sh
```

Windows PowerShell:

```powershell
irm https://raw.githubusercontent.com/JegernOUTT/refact/main/install.ps1 | iex
```

Start Refact with:

```sh
refact
refact tui
```
