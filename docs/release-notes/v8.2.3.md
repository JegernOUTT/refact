# Refact v8.2.3

Refact v8.2.3 adds the CodeGraph source-code indexing subsystem and surfaces its status across the engine, GUI, and IDE hosts while preserving VecDB for memory-plane semantic search.

> [!WARNING]
> **Breaking change — local daemon**
> The plugins require and launch a local `refact daemon` (`127.0.0.1:8488`) that supervises per-project workers. The engine binary is downloaded on first run from GitHub releases and is no longer bundled in the plugin; the in-IDE webview connects over loopback. If you used a pre-8.2.0 bundled-engine version, first start now downloads the engine.

> [!IMPORTANT]
> **Early preview**
> The daemon and TUI are early preview and may be buggy. IDE plugins and the engine are the stable surfaces.

## Highlights

- CodeGraph now owns source-code indexing in a persistent SQLite store under `~/.cache/refact/codegraph/`, with symbol/reference extraction, FTS search, and graph analysis backed by the `refact-codegraph`, `refact-codegraph-parsers`, `refact-codehealth`, `refact-codewiki`, and `refact-git-intel` crates.
- New CodeGraph-powered surfaces include `search_symbol_definition`, `codegraph_overview`, `code_health`, `git_risk`, `code_why`, `code_duplication`, `code_map`, `/v1/codegraph-status`, and `/v1/codegraph-search`.
- `/v1/rag-status` now reports CodeGraph status as `{ counts: { nodes, edges, files, fts_docs }, queued, state, error }` plus `codegraph_alive` and `codegraph_error`; valid states are `turned_off`, `indexing`, `working`, and `error`.
- Workspace indexing now routes through a memory-plane firewall: knowledge and trajectory paths go to VecDB, while source-code paths go to CodeGraph so code retrieval does not pollute memory search.
- The GUI toolbar and IDE status bars surface CodeGraph indexing progress, queued files, counts, and errors alongside existing VecDB/AST compatibility status.

## Install

Unix/macOS:

```sh
curl -fsSL https://raw.githubusercontent.com/JegernOUTT/refact/main/install.sh | sh
```

Windows PowerShell:

```powershell
irm https://raw.githubusercontent.com/JegernOUTT/refact/main/install.ps1 | iex
```

Start Refact with:

```sh
refact
refact tui
```
