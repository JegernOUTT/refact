# Refact v8.2.0

Refact v8.2.0 moves engine delivery to download-on-first-run, adds a resident local daemon with warm project workers, and tightens VS Code/JetBrains connection reliability. This release also adds the Refact Daemon page and brings JetBrains and VS Code behavior closer to parity.

> [!WARNING]
> **Breaking change — local daemon**
> The plugins now require and launch a local `refact daemon` (`127.0.0.1:8488`) that supervises per-project workers. The engine binary is downloaded on first run from GitHub releases and is no longer bundled in the plugin; the in-IDE webview connects over loopback. If you used a prior bundled-engine version, first start now downloads the engine.

> [!IMPORTANT]
> **Early preview**
> The daemon and TUI are early preview and may be buggy. IDE plugins and the engine are the stable surfaces.

## Highlights

- Download-only engine delivery: IDE plugins fetch the matching `refact` engine on first run instead of bundling binaries.
- Resident daemon + warm workers: one local daemon supervises project workers and keeps projects ready across VS Code, JetBrains, TUI, and browser GUI clients.
- VS Code and JetBrains connection fixes: health probes, startup/status reporting, version gates, timeout parity, and stale-lock recovery are more consistent across IDEs.
- New Refact Daemon page: inspect daemon health, workers, projects, logs, events, and cron wakeups from the GUI.
- JetBrains ↔ VS Code behavior parity: daemon attach, project proxy URLs, browser endpoint handling, and engine download behavior now follow the same release path.

## Install

Unix/macOS:

```sh
curl -fsSL https://raw.githubusercontent.com/JegernOUTT/refact/main/install.sh | sh
```

Windows PowerShell:

```powershell
irm https://raw.githubusercontent.com/JegernOUTT/refact/main/install.ps1 | iex
```

Start Refact with:

```sh
refact
refact tui
```
