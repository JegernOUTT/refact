# v8.2.7

## Compatibility & release process

- **JetBrains plugin passes 2026.2 verifier with zero findings.** JCEF was moved out of
  the core classpath into the `com.intellij.modules.jcef` plugin in 2026.2; the plugin
  now declares an optional dependency and uses a `NoClassDefFoundError`-safe probe so
  JCEF-less IDEs degrade gracefully instead of crashing at startup.
- **Double action registration fixed** for IntelliJ 2026.2's stricter action registrar.
- **Engine release workflow rebuilt** to build through the same reusable CI pipeline
  that plugin releases always used, eliminating cross-matrix divergence.

## Codex (OpenAI Responses API)

- `codex.response.metadata` events (safety-buffering headers) and any future unknown
  `codex.*` stream events are now absorbed as message metadata instead of rendering
  an unhandled server block in the chat.
- `gpt-5.6-luna` is a server-internal safety-buffering draft model — advertised by
  `/codex/models` but rejected by `/codex/responses` (404). It is no longer
  selectable in the model picker.

## Engine reliability

- Buddy diagnostics persistence test deflaked: polls the file instead of sleeping
  a fixed 150ms (fixes aarch64-windows CI flake).
- Goal-validator messages no longer block the generation stream after validation
  dismisses them.

## Daemon binary resolution (IDE plugins)

- Both plugins now self-identify correctly for their version (no hardcoded fallbacks).
- When a pinned engine release has not been published yet (e.g. fresh release tag
  between cut and upload), the plugin falls back to the latest published engine
  release and shows a one-time warning instead of silently reusing a stale engine
  or failing with a cryptic error.
