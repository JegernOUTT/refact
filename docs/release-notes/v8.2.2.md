# Refact v8.2.2

Refact v8.2.2 is a hotfix for v8.2.1: the IDE plugins could show "Connection Problem" because the local daemon's LAN-auth hardening rejected the plugin webviews' cross-origin requests. This release restores connectivity and makes releases publish the engine before the plugins.

> [!WARNING]
> **Breaking change — local daemon**
> The plugins require and launch a local `refact daemon` (`127.0.0.1:8488`) that supervises per-project workers. The engine binary is downloaded on first run from GitHub releases and is no longer bundled in the plugin; the in-IDE webview connects over loopback. If you used a pre-8.2.0 bundled-engine version, first start now downloads the engine.

> [!IMPORTANT]
> **Early preview**
> The daemon and TUI are early preview and may be buggy. IDE plugins and the engine are the stable surfaces.

## Fixes

- Daemon auth/CORS: trust the IDE webview origins (`http://refactai` for JetBrains, `vscode-webview://<uuid>` for VS Code) on loopback, so the in-IDE chat connects again. Remote/LAN access still requires authentication; unknown web origins are still rejected.
- Release pipeline: publish the engine release (`engine/v{version}`) before the IDE plugins, so a freshly-installed plugin can always download its matching engine (no more "engine release unavailable" right after publish).

## Install

Unix/macOS:

```sh
curl -fsSL https://raw.githubusercontent.com/JegernOUTT/refact/main/install.sh | sh
```

Windows PowerShell:

```powershell
irm https://raw.githubusercontent.com/JegernOUTT/refact/main/install.ps1 | iex
```

Start Refact with:

```sh
refact
refact tui
```
